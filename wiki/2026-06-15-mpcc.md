# MPCC

## 基本信息

- 全称：Model Predictive Contouring Control
- 类型：自主赛车预测控制研究参考实现
- 作者：Alexander Liniger
- 研究机构背景：ETH Zurich Automatic Control Laboratory
- 官方仓库：https://github.com/alexliniger/MPCC
- 主要语言：C++14、MATLAB
- 核心求解器：HPIPM
- 许可证：Apache License 2.0
- 默认分支：`master`
- 快照提交：`bd331621ba47ae3326711922a863bdb1cdf2d2ea`
- 最后提交时间：2021-11-11

## 当前判断

MPCC 是理解“将路径进度作为优化变量”的自主赛车控制器的优质源码。它适合研究 contouring/lag error、弧长参考线、动态自行车模型、轮胎约束、SQP 实时迭代和 OCP-QP 求解，不适合未经重构直接部署到全尺寸车辆或道路自动驾驶系统。

## 控制问题

```text
赛道中心线与边界
+ 车辆动态状态
+ 上一周期预测轨迹
→ 线性化动力学、误差和约束
→ HPIPM 求解时变 QP
→ SQP 解混合与 warm start
→ 第一拍驱动率、转角率和进度率
```

优化目标同时考虑：

- contouring error
- lag error
- 路径进度
- 输入变化率
- 横摆角速度和质心侧偏角正则

约束包括：

- 状态和输入边界
- 输入变化率
- 赛道半空间
- 前轮侧偏角
- 后轮摩擦椭圆
- 路径进度 trust region

## 实现差异

### C++

- 10 维状态、3 维输入、60 步预测。
- 连续时间输入变化率模型。
- 每周期多轮 SQP，默认两轮。
- 仅支持 HPIPM。
- 增加轮胎约束和侧偏角代价。
- 不支持避障。
- `master` 不适合全尺寸车参数。

### MATLAB

- 更接近原始 1:43 赛车论文。
- 支持 HPIPM、YALMIP、CVX 和 `quadprog`。
- 支持 ORCA 与 FullSize 模型。
- 使用一维动态规划选择避障走廊。

## 最值得复用

- 将路径弧长进度引入 MPC 状态。
- contouring 与 lag error 的局部二次化。
- 赛道边界的线性半空间表达。
- warm start、归一化和 SQP 迭代结构。
- 轮胎侧偏角与摩擦椭圆约束。
- 求解失败后的初值重置思路。

## 工程边界

- 最后更新于 2021 年。
- 外部依赖不锁版本。
- 文档依赖历史 Python 2.7 环境。
- 测试函数被编译但默认不运行。
- 无 CI、安装目标和稳定库 API。
- 固定混合 SQP 没有 line search 或 merit function。
- 低速动态模型需要特殊处理。
- `master`、`main` 和 `full-size` 分支语义需分别核对。
