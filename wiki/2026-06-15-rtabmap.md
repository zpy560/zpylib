# RTAB-Map

## 基本信息

- 全称：Real-Time Appearance-Based Mapping
- 类型：多传感器图优化 SLAM 库与独立应用
- 维护者：Mathieu Labbé / IntRoLab
- 官方仓库：https://github.com/introlab/rtabmap
- 项目主页：https://introlab.github.io/rtabmap/
- 主要语言：C++、CMake、Qt、Java、Swift
- 许可证：BSD 3-Clause
- 快照分支：`master`
- 快照提交：`9f464db7c9d87867cb0dad98871a1614db0b62cf`
- 快照日期：2026-06-15
- 分析方式：本地静态源码分析，未编译、未运行

## 当前判断

RTAB-Map 是长期运行机器人 SLAM 的高价值参考实现。其核心不是某个视觉特征，而是把有界工作记忆、可召回长期数据库、多级回环验证和位姿图优化组织成完整系统。

它的功能覆盖和工具链成熟，但配置与依赖复杂、核心主循环集中、自动测试薄弱。工程使用必须固定编译能力和运行参数，并以数据库和数据集评测验证实际行为。

## 架构总览

```text
Sensor Capture
→ Odometry
→ Signature / Memory
→ Loop Closure and Proximity Detection
→ Registration
→ Graph Optimization
→ Map Correction
→ Occupancy / Cloud / OctoMap / Mesh
→ SQLite Database
```

## 核心模块

- `Rtabmap`：SLAM 主循环、模式和全局状态。
- `Memory`：Signature、视觉词、STM/WM/LTM 和数据库调度。
- `Odometry`：视觉、RGB-D、LiDAR 和 VIO 里程计工厂。
- `Registration`：视觉、ICP 和级联节点注册。
- `Optimizer`：g2o、GTSAM、TORO、Ceres 与 BA。
- `DBDriverSqlite3`：地图、原始数据、特征、约束与统计持久化。
- `OccupancyGrid` / `CloudMap` / `OctoMap`：地图生成。
- `RtabmapThread` / `OdometryThread`：独立应用事件流水线。

## 核心能力

- RGB-D、双目、单目和 LiDAR SLAM。
- 外观回环、空间邻近回环和多会话重定位。
- 2D 与 3D 图优化。
- 视觉、ICP、视觉惯性和激光惯性里程计接入。
- 2D 栅格、点云、OctoMap、网格和纹理地图。
- SQLite 地图数据库与离线重处理。
- Linux、Windows、macOS、Android、iOS 和 ROS 构建。

## 关键设计

### 分层记忆

STM 保护最新节点，WM 控制在线计算规模，LTM 保存长期历史。历史区域成为回环候选时按需从数据库重激活。

### 约束优先的数据模型

节点保存观测，`Link` 保存邻接、回环、地标、重力和虚拟约束。地图是节点与约束优化的结果，不是不可追溯的最终点云。

### 在线与离线共用数据库

在线建图生成 `.db`；离线工具可补回环、重新优化、修复、导出和评测，降低重新采集成本。

## ROS 与 Navigation2

本仓库提供底层库；ROS 节点和 launch 主要属于 `rtabmap_ros`。

在 ROS 2 系统中：

```text
RTAB-Map / rtabmap_ros
→ map、map→odom、定位与栅格
→ Navigation2
→ 规划、控制、恢复与执行
```

## 主要风险

1. 编译选项和外部后端很多，实际能力依赖构建环境。
2. 参数工厂可能自动回退，配置名称不等于真实运行实现。
3. `Rtabmap::process()` 职责集中，修改回归风险较高。
4. SQLite 异步写入的线程模型有待重构注释。
5. README、package 和 CMake 版本号不一致。
6. CI 主要验证编译，没有正式单元测试和数据集回归。
7. 回环和图优化正确性仍依赖标定、噪声和阈值。

## 推荐评估

- TUM RGB-D：RGB-D SLAM 与回环。
- KITTI：视觉/激光里程计和较大场景。
- EuRoC：双目惯性和 VIO。
- 自有数据：传感器同步、动态环境、重复纹理和长期重定位。
