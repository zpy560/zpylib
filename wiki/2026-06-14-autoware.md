# Autoware

## 基本信息

- 类型：ROS 2 原生的开源自动驾驶软件平台
- 官方仓库：https://github.com/autowarefoundation/autoware
- 主要语言：C++、Python、XML、YAML
- 主仓库提交：`4847819760656247343dfb995ea8e151fc3d9b00`
- Core 版本：`1.8.0`
- Universe 版本：`0.51.0`
- Launch 版本：`0.51.0`
- 快照日期：2026-06-24
- 分析方式：本地静态分析，未编译、未运行

## 当前判断

Autoware 是研究 ROS 2 如何承载完整自动驾驶栈的高价值参考。它最值得学习的是版本化多仓库装配、规划插件流水线、车辆与传感器模型隔离，以及控制命令之后的门控和检查体系。

主要工程风险来自分布式配置面：manifest、launch、参数、remap、组件容器和功能开关共同决定最终行为。模块源码存在不等于部署中已经启用。

## 架构总览

```text
Sensing
→ Localization / Map
→ Perception
→ Mission and Scenario Planning
→ Planning Validator
→ Trajectory Follower
→ Command Gate
→ Vehicle Interface
```

## 规划架构

规则规划按 Lane Driving 与 Parking 分流。Lane Driving 继续拆分为行为路径、行为速度、路径平滑/优化、运动速度和速度平滑，各层通过插件组合道路业务与运动约束。

学习型 Diffusion Planner 可以替代规则 Scenario Planning，但仍输出统一轨迹并经过 Planning Validator。

## 控制与安全

- 默认横向 MPC、纵向 PID。
- Shift Decider 生成挡位请求。
- Command Gate 处理自动、外部和紧急命令。
- Control Validator 检查控制输出与车辆状态。
- Lane Departure Checker、AEB、Collision Detector 提供独立监控。
- Operation Mode 与 MRM 管理可用性和降级状态。

`vehicle_cmd_gate` 与 `control_command_gate` 当前存在迁移关系，部署审计必须确认实际启用路径。

## 对规划控制的启发

1. 用统一轨迹消息隔离规划策略与控制器。
2. 将道路业务规则、路径生成和速度约束拆成不同插件层。
3. 版本清单必须与 launch 和参数一起纳入代码评审。
4. 控制命令必须经过模式管理和独立门控。
5. 学习型规划应复用既有输出契约和后置验证。
6. 安全分析从车辆执行接口反向检查真实接线。
