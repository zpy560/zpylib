## 项目定位

planning_algorithm 是一个极简 C++ 路径规划教学仓库，实际实现只有：

- 二维栅格 A*
- 二维圆形障碍环境 RRT

README 中的大篇幅表格介绍自动驾驶运动规划分类，包括 Dijkstra、A* 家族、State Lattice、RRT、插值曲线和数值优化，但这些分类不代表仓库实现范围。

## A* 实现

### 地图与状态

- 固定 50 x 50 栅格。
- 每格绘制为 10 x 10 像素。
- 障碍物以硬编码矩形写入。
- 起点固定为 `(10, 40)`，终点固定为 `(0, 0)`。
- 邻居为八连通栅格。
- 直移代价为 1，对角代价为 `sqrt(2)`。
- 启发式为到终点的欧氏距离。

搜索使用 `std::priority_queue<Node*>` 按 `g + h` 排序，并用 OpenCV 动态绘制访问节点和最终路径。

### 与文档伪代码的偏差

文档正确描述了：

```text
若节点已在 open set 且发现更小 g
→ 更新 backpointer 和代价
```

源码没有实现该松弛逻辑。节点第一次加入队列后立即设置 `is_visited`，后续发现同一节点时直接跳过。因此算法采用“首次发现即确定父节点”，不保证在一般加权图或不一致启发式下获得正确最短路径。

其他问题：

- 起点在压入 open set 前就加入 closed set。
- 搜索函数无论是否找到目标都返回 `true`。
- 失败时 `DrawPath()` 可能从空父指针开始回溯。
- 对角移动没有检查相邻正交格，允许穿过障碍物角点。
- 障碍物写入没有边界校验。
- 全局地图状态不支持重复运行。
- `main()` 成功结束返回 `-1`。

在当前固定地图和欧氏启发式下可能得到一条可视路径，但实现不应作为通用 A* 参考。

## RRT 实现

### 搜索过程

- 工作区域固定为 `[0, 15] x [0, 15]`。
- 起点固定为 `(2, 2)`，终点固定为 `(14, 9)`。
- 障碍物是硬编码圆形集合。
- 每次从均匀随机位置或终点采样。
- 线性扫描整棵树寻找最近节点。
- 沿采样方向前进一步生成新节点。
- 新节点进入终点一步范围后结束。

OpenCV 绘制障碍物、随机树和最终父节点链。

### 正确性与鲁棒性边界

- 碰撞只检查新节点是否落在圆障碍内，不检查父节点到新节点的线段，可能穿越障碍。
- 达到终点阈值后没有把最后节点与目标点做碰撞连接。
- 返回路径把目标点直接放到列表头，但目标节点没有接入树。
- 没有最大迭代数，困难或无解场景可能无限运行。
- 随机种子来自 `random_device`，不能复现实验。
- 采样时误用 `goal_gen_` 驱动区域分布，单独创建的 `area_gen_` 没有使用。
- 节点使用裸 `new`，没有释放。
- 没有机器人半径或障碍膨胀。
- 没有 RRT* 重布线、代价最优性或路径平滑。
