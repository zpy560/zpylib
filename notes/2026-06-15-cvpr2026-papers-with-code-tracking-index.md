# CVPR2026-Papers-with-Code：持续更新的论文与代码追踪目录

## 一句话结论

该仓库是一个由社区持续补充的 CVPR 2026 论文、代码和项目页导航，适合发现已公开项目并按方向跟踪研究动态；当前内容远未覆盖全部录用论文，分类、链接完整性和收录标准也不统一，不能替代 OpenReview、CVF 官方论文集或结构化文献数据库。

## 原文要点

### 仓库定位

- README 声称 CVPR 2026 共收到 16092 篇投稿，录用 4090 篇，录用率为 25.42%。
- 仓库欢迎作者或读者通过 Issue 和 Pull Request 补充论文与开源项目。
- 内容按研究方向组织，每项通常提供 Paper、Code、Project 或 Model 链接。
- 仓库同时保留 CVPR 2019 至 2025 的历年列表，具备跨年度追踪入口。

### 当前快照覆盖

本次 README 快照共 600 行，目录列出约 80 个方向，但正文只有 27 个明确论文标题、30 个论文链接、19 个有效代码链接和 8 个项目页链接。大量分类只有标题没有条目。

已有内容主要分布在：

- 3D Gaussian Splatting 与 3D 重建
- 多模态大模型、视觉语言与空间智能
- 具身智能
- 医学图像分割
- 自动驾驶与 3D 点云
- 图像生成、遥感、视频压缩、图像检索与鲁棒水印

自动驾驶分类当前只列出两项：

1. 城市场景分割的开放词汇域泛化。
2. 基于 LiDAR 序列的不确定性感知 4D 世界建模。

### 数据结构与维护方式

仓库本质上是单个 README 驱动的人工目录，没有统一元数据 schema。不同条目可能缺少代码、项目页或论文链接，同一论文也可能在多个方向重复出现。

## 我的判断

### 适合作为发现入口，不适合作为完整数据源

这个仓库的核心价值是降低“论文是否已经有代码或项目页”的查找成本，尤其适合快速浏览新方向和定位可运行项目。它不具备完整性保证，也没有清晰记录收录时间、最后核验时间、官方录用状态和代码可复现性。

正确用法是：

```text
仓库发现候选论文
→ OpenReview 或 CVF 核验录用与版本
→ 论文主页核验作者和材料
→ 代码仓库检查许可证、提交活跃度和复现说明
→ 选出值得深入阅读或实验的项目
```

### 对自动驾驶研究的价值有限但直接

目录覆盖感知、3D、世界模型、轨迹预测和车道线等自动驾驶相关方向，适合作为感知与生成式世界模型的增量雷达。但当前自动驾驶正文只有两项，不能据此判断 CVPR 2026 自动驾驶研究全貌，更不能直接形成技术趋势结论。

### 主要风险

1. README 处于快速更新阶段，今天的统计很快会失效。
2. 分类目录与正文不完全一致，部分锚点、括号和标题格式存在错误。
3. “Papers with Code”只是仓库名称，部分条目没有代码链接。
4. 社区提交机制提高覆盖速度，也带来重复、误分类和录用状态待核验的问题。
5. 仓库热度不能代表单篇论文质量，代码链接存在也不能代表可复现。

## 可复用内容

### 论文筛选清单

1. 是否能在 OpenReview 或 CVF 核验正式录用状态？
2. 论文版本、项目页和代码仓库是否属于同一作者团队？
3. 代码是否提供许可证、环境、权重、数据和评测脚本？
4. 最近提交是否活跃，Issue 中是否存在无法复现的关键问题？
5. 方法是否在与自身任务一致的数据集和指标上有效？
6. 对自动驾驶任务，是否覆盖闭环、实时性、长尾和安全约束？

## 疑问与冲突

- README 中的录用统计需要以 CVPR/OpenReview 官方信息为最终依据。
- 当前快照明显处于早期收集阶段，后续应按需要重新抓取，而不是把本次数量当作稳定结论。
- 仓库没有提供统一的论文唯一标识，跨分类去重和跨年度分析需要额外整理。
